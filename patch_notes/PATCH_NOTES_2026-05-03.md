# 패치 노트

날짜: 2026-05-03
프로젝트: LOL Team Builder

## 요약

이번 작업에서는 최근 전적 기반 팀 빌더 로직을 보강하고, 저장 계정 조회와 최근 경기 분석 흐름을 다듬었습니다. 또한 클라이언트 구조를 `application`, `api_clients`, `domain`, `repositories`, `ui`로 재정리했고, 데이터셋 관리 UX와 다크/화이트 테마 설정을 추가했습니다. 마지막으로 Riot API를 이용해 계정의 티어 정보를 함께 적재하고, `선택 유저 추가` 시 자동으로 반영되도록 연결했습니다.

## 상세 변경 사항

### 1. 팀 생성 계산 로직 보강

- 최근 경기 결과를 반영한 최근 폼 보너스 계산을 추가했습니다.
- 최근 포지션별 경기 비중과 승률을 반영한 포지션 적합 보너스 계산을 추가했습니다.
- 팀 점수 계산 시 티어 점수 외에 최근 승률, KDA, 포지션 적합도를 함께 반영하도록 조정했습니다.
- 라인별 점수 차이, 최대 라인 차이, 원딜-서포터 차이, 부계정 격차를 함께 비교하도록 후보 선정 우선순위를 보강했습니다.
- 조건을 완전히 만족하는 조합이 없을 때 완화 규칙으로 다시 조합을 시도하는 fallback 흐름을 추가했습니다.

### 2. 최근 전적 분석 및 유저 프로필 생성 정리

- Riot 계정 검색 기능을 클라이언트 메인 흐름에서 바로 사용할 수 있도록 정리했습니다.
- 최근 경기 목록, 선택 경기 상세, 포지션 통계, 챔피언 통계 UI 흐름을 연결했습니다.
- 최근 승률, KDA, 평균 CS, 시야 점수, 피해량, 골드 등의 요약 데이터를 계산하는 애플리케이션 계층을 구성했습니다.
- 선택한 계정을 유저 표에 추가할 때 최근 경기 기반 수치와 메타 데이터를 함께 저장하도록 구성했습니다.
- 팀 결과 화면에서 각 유저의 최근 폼과 포지션 적합도를 볼 수 있도록 정리했습니다.

### 3. Riot 적재 도구 개선

- `client/tools/riot_loader.py`를 현재 클라이언트 구조에 맞게 정리했습니다.
- puuid 조회, 매치 ID 조회, 첫 경기 상세 조회, 최근 경기 DB 적재 흐름을 유지하면서 코드 구조를 정리했습니다.
- 저장된 계정을 검색하고 여러 계정을 선택해 일괄 적재하는 흐름을 유지했습니다.
- 저장된 계정 전체 목록을 한 번에 불러오는 기능을 추가했습니다.
- Riot 로더에서 검색 개수 입력 UI를 제거하고, 검색 제한은 공통 상수로 통일했습니다.
- 서버에는 저장 계정 전체 목록 조회용 `/accounts` 엔드포인트를 추가했습니다.

### 4. 클라이언트 디렉터리 구조 개편

- 루트에 흩어져 있던 클라이언트 관련 파일을 `client/` 아래로 정리했습니다.
- 기존 `service` 계층을 다음과 같이 역할별로 분리했습니다.
- `client/api_clients/`: 서버 API 통신
- `client/domain/`: 팀 계산 로직과 도메인 규칙
- `client/repositories/`: 로컬 데이터와 설정 파일 입출력
- 기존 `ui/presenter.py`를 제거하고 `client/application/` 아래로 역할을 분리했습니다.
- `account_queries.py`
- `match_analysis.py`
- `team_building.py`
- `table_data.py`
- `result_formatters.py`
- `recent_match_views.py`
- `team_app.py`
- `main_window.py`는 화면 조립과 이벤트 처리 위주로 줄이고, 최근 경기 요약과 행 생성 로직은 `application/recent_match_views.py`로 분리했습니다.

### 5. 데이터셋 관리 UX 개선

- 데이터셋 생성 외에 데이터셋 복사 기능을 추가했습니다.
- 데이터셋 삭제 기능을 추가했고, 삭제 전 확인 팝업이 반드시 표시되도록 했습니다.
- `선택 유저 추가`는 데이터셋이 실제로 불러와진 상태에서만 동작하도록 제한했습니다.

### 6. 계정 검색 UX 정리

- 메인 클라이언트에서 최근 전적 불러오기 버튼을 제거했습니다.
- 계정 검색 시 첫 계정을 자동 선택하고 최근 전적도 함께 불러오도록 흐름을 단순화했습니다.
- 검색창에 포커스가 있을 때 `Enter`를 누르면 계정 검색이 실행되도록 연결했습니다.
- 팀 결과 위젯에서 계정 정보가 있는 유저를 클릭하면 해당 계정 검색이 열리도록 연결했습니다.
- 메인 클라이언트의 검색 결과 개수 입력 UI를 제거했습니다.
- 계정 검색 결과 제한은 `ACCOUNT_SEARCH_LIMIT = 1000` 상수로 고정했습니다.
- 서버의 `/accounts/search` 최대 조회 개수도 `1000`으로 맞췄습니다.

### 7. 상수와 UI 표현 분리

- 기존 `client/util/constants.py`를 제거했습니다.
- 도메인 규칙과 기본 상수는 `client/domain/constants.py`로 이동했습니다.
- 티어 색상, 최근 전적 요약 스타일, 테마 토큰은 `client/ui/theme.py`로 분리했습니다.
- 각 계층에서 필요한 상수만 직접 참조하도록 import 경로를 정리했습니다.

### 8. 다크 모드 / 화이트 모드 추가

- 설정 팝업에 `다크 모드`, `화이트 모드` 선택 기능을 추가했습니다.
- 테마 설정값을 `config.json`에 저장하도록 구성했습니다.
- 설정 저장 후 메인 화면에 선택한 테마가 즉시 다시 적용되도록 연결했습니다.
- 기존 `dark.qss`를 정리하고 `light.qss`를 새로 추가했습니다.
- 스타일 로더가 저장된 테마 설정값에 따라 적절한 QSS를 불러오도록 변경했습니다.
- 최근 전적 요약 영역, 유저 테이블, 팀 결과 위젯의 하드코딩 색상을 테마 기반 토큰으로 분리했습니다.

### 9. 라이엇 티어 정보 적재 연동

- Riot 공식 API의 `summoner-v4`, `league-v4`를 이용해 계정의 랭크 정보를 함께 가져오도록 서버 적재 흐름을 확장했습니다.
- `store_recent_matches` 실행 시 Riot ID 조회 이후 소환사 정보와 랭크 엔트리를 추가로 조회하도록 구성했습니다.
- 저장 계정 테이블인 `riot_account`에 `summoner_id`, `summoner_level`, `profile_icon_id`, `queue_type`, `tier`, `rank`, `league_points`, `wins`, `losses` 컬럼을 저장하도록 확장했습니다.
- 기존 DB도 서버 시작 또는 조회 시 자동으로 컬럼이 보강되도록 마이그레이션 로직을 추가했습니다.
- 저장 계정 조회 API 응답에 티어 관련 필드가 포함되도록 `match_read_query`와 `match_service`를 확장했습니다.
- 클라이언트의 `선택 유저 추가` 시 검색 결과에 포함된 `tier`, `rank`를 읽어 자동으로 `tier`, `tier_detail`에 반영하도록 연결했습니다.
- `rank` 값은 `I -> 1`, `II -> 2`, `III -> 3`, `IV -> 4`로 매핑해 기존 밸런싱 로직과 호환되도록 처리했습니다.

### 10. Git 추적 및 산출물 관리 정리

- `.gitignore`에서 `dist/` 전체를 무시하도록 정리했습니다.
- `client/team_builder_client_error.log`를 Git 추적 대상에서 제외했습니다.
- 클라이언트 데이터 JSON은 기본적으로 Git 추적 대상에서 제외되도록 정리했습니다.
- 기존에 추적 중이던 `dist/` 내부 파일도 저장소에서 제외되도록 정리했습니다.

### 11. 문서 및 패치노트 체계 정리

- `README.md`를 현재 구조 기준으로 다시 작성했습니다.
- 클라이언트, Riot 로더, 서버, PyInstaller 실행 명령을 README에 정리했습니다.
- 패치노트를 `patch_notes/` 디렉토리에서 날짜별로 관리하는 구조로 정리했습니다.

## 영향

- 클라이언트 코드의 책임 구분이 더 분명해져 이후 유지보수와 기능 추가가 쉬워졌습니다.
- 저장 계정 검색과 최근 전적 조회 흐름이 더 직관적으로 바뀌었습니다.
- 데이터셋 관리와 선택 유저 추가 흐름이 더 안전해졌습니다.
- 다크 모드와 화이트 모드를 프로젝트 설정 수준에서 전환할 수 있게 되어 사용 환경 선택 폭이 넓어졌습니다.
- Riot 로더와 메인 클라이언트의 검색 동작 기준이 일치해 운영 시 혼란이 줄어들었습니다.
- 저장 계정에 티어 정보가 함께 누적되므로 이후 유저 추가 시 수동 티어 입력 부담이 줄어듭니다.
