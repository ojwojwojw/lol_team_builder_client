# 패치 노트

날짜: 2026-05-03
프로젝트: LOL Team Builder

## 요약

이번 작업에서는 최근 전적 기반 분석 흐름과 팀 생성 보정 로직을 정리하고, 클라이언트 구조를 `application`, `api_clients`, `domain`, `repositories`, `ui`로 재분리했습니다. 또한 저장 계정 전체 불러오기 기능을 Riot 로더에 추가했고, 문서와 패치노트 체계도 함께 정리했습니다.

## 상세 변경 사항

### 1. 팀 생성 계산 로직 보강

- 최근 경기 수를 바탕으로 신뢰도를 반영한 `최근 폼 보너스` 계산을 추가했습니다.
- 최근 포지션별 경기 비중과 승률을 바탕으로 `포지션 적합도 보너스` 계산을 추가했습니다.
- 팀 전투력 계산 시 티어 점수 외에 최근 승률, KDA, 포지션 적합도를 함께 반영하도록 조정했습니다.
- 라인별 전투력 차이, 최대 라인 차이, 원딜 맞라인 차이, 불균형 심각도를 함께 비교하도록 후보 선정 우선순위를 보강했습니다.
- 조건을 완전히 만족하는 조합이 없을 때 완화 규칙으로 다시 조합을 시도하는 fallback 흐름을 유지하도록 구성했습니다.

### 2. 최근 전적 분석 및 유저 프로필 생성 기능 정리

- Riot 계정 검색 기능을 클라이언트 흐름에서 사용할 수 있도록 정리했습니다.
- 최근 경기 목록, 선택 경기 상세, 포지션 통계, 챔피언 통계 UI 흐름을 유지 가능한 구조로 정리했습니다.
- 최근 승률, KDA, 평균 CS, 시야 점수, 딜량, 골드 등의 요약 데이터를 계산하는 애플리케이션 계층을 구성했습니다.
- 선택한 계정을 유저 표에 추가할 때 최근 경기 기반 포지션과 메타 데이터를 함께 저장하도록 구성했습니다.
- 팀 결과 화면에 각 유저의 최근 폼과 포지션 적합도를 표시하도록 정리했습니다.

### 3. Riot 적재 도구 개선

- `client/tools/riot_loader.py`를 기준으로 수동 적재 도구를 클라이언트 패키지 내부로 유지하면서 역할을 정리했습니다.
- puuid 조회, 매치 ID 조회, 첫 경기 상세 조회, 최근 경기 DB 적재 흐름을 현재 구조에 맞게 정리했습니다.
- 저장된 계정을 검색하고 여러 계정을 선택해 일괄 적재하는 흐름을 계속 사용할 수 있도록 정리했습니다.
- 저장된 계정 전체 목록을 한 번에 불러오는 기능을 추가했습니다.
- 서버에는 저장 계정 전체 목록 조회용 `/accounts` 엔드포인트를 추가했습니다.

### 4. 클라이언트 디렉토리 구조 재편

- 루트에 흩어져 있던 클라이언트 파일을 모두 `client/` 아래로 이동했습니다.
- 기존 `service` 계층을 다음과 같이 역할별로 분리했습니다.
  - `client/api_clients/`: 서버 API 통신
  - `client/domain/`: 팀 계산 로직과 도메인 규칙
  - `client/repositories/`: 로컬 데이터와 설정 파일 입출력
- 기존 `ui/presenter.py`를 제거하고 `client/application/` 아래로 역할을 분리했습니다.
  - `account_queries.py`
  - `match_analysis.py`
  - `team_building.py`
  - `table_data.py`
  - `result_formatters.py`
  - `recent_match_views.py`
  - `team_app.py`
- `main_window.py`는 화면 조립과 이벤트 처리 위주로 줄이고, 최근 경기 관련 row 생성과 요약 문구 생성은 `application/recent_match_views.py`로 분리했습니다.

### 5. 상수와 UI 표현 분리

- 기존 `client/util/constants.py`를 제거했습니다.
- 도메인 규칙과 데이터는 `client/domain/constants.py`로 이동했습니다.
- 티어 색상과 최근 요약 스타일 같은 UI 표현 상수는 `client/ui/theme.py`로 분리했습니다.
- 각 위젯과 계산 로직이 필요한 계층에서만 상수를 직접 참조하도록 import 경로를 정리했습니다.

### 6. Git 추적 및 산출물 관리 정리

- `.gitignore`에서 `dist/` 전체를 무시하도록 유지했습니다.
- `client/team_builder_client_error.log`를 Git 추적 대상에서 제외했습니다.
- 클라이언트 데이터 JSON은 기본적으로 Git 추적 대상에서 제외되도록 유지했습니다.
- 기존에 추적 중이던 `dist/` 내부 파일은 저장소에서 제외되도록 정리했습니다.

### 7. 문서 갱신

- `README.md`를 현재 구조 기준으로 한글화하고 다시 작성했습니다.
- 클라이언트, Riot 로더, 서버, PyInstaller 실행 명령을 README에 정리했습니다.
- 패치노트를 `patch_notes/` 디렉토리에서 날짜별로 관리하도록 기준을 유지했습니다.

## 영향

- 클라이언트 구조가 역할별로 분리되어, 이후 기능 추가나 테스트 범위 구분이 쉬워졌습니다.
- `service`와 `presenter`처럼 역할이 넓게 섞여 있던 파일들이 줄어들어 코드 탐색성이 좋아졌습니다.
- 상수와 UI 표현이 분리되어, 도메인 규칙 수정과 화면 표현 수정을 서로 덜 건드리게 되었습니다.
- 저장 계정 전체 목록을 바로 불러올 수 있어 여러 명의 전적 적재 작업이 더 편해졌습니다.
