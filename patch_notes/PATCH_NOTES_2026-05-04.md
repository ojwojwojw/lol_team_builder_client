# 패치 노트

날짜: 2026-05-04
프로젝트: LOL Team Builder

## 요약

이번 작업에서는 Riot 적재 도구에 관리자 전용 로그인 흐름을 연결하고 UI를 정리했습니다. 또한 서버 JWT 구현을 `PyJWT` 기반으로 교체했고, 최근 경기 적재 시 이미 DB에 있는 매치를 `match_id` 기준으로 먼저 확인한 뒤 신규 경기만 Riot 상세 조회를 호출하도록 최적화했습니다.

## 상세 변경 사항

### 1. Riot 로더 관리자 로그인 적용

- `client/tools/riot_loader.py` 실행 시 기존 JWT 토큰을 확인하고, 필요하면 로그인 다이얼로그를 통해 다시 인증하도록 연결했습니다.
- 관리자 계정이 아닌 경우 Riot 로더 진입을 막고, 관리자 계정으로 다시 로그인하도록 안내합니다.
- Riot 로더의 모든 요청에 `Authorization: Bearer <token>` 헤더가 붙도록 정리했습니다.

### 2. Riot 로더 UI 개선 및 한글화

- 상단에 관리자 세션, 서버 주소, 재로그인 버튼을 표시하는 상태 영역을 추가했습니다.
- 수동 적재, 저장 계정 일괄 적재, 응답 결과 영역을 그룹 박스로 분리해 화면 구조를 정리했습니다.
- 버튼, 라벨, 플레이스홀더, 상태 메시지를 한글 기준으로 정리해 운영 시 읽기 쉽도록 맞췄습니다.
- 클라이언트 공용 테마 스타일을 Riot 로더에도 적용하도록 연결했습니다.

### 3. Riot 적재 API 관리자 권한 강제

- `server/backend/controllers/riot_controller.py` 전체에 관리자 권한 의존성을 적용했습니다.
- 이제 Riot 관련 조회/적재/티어 갱신 API는 로그인만으로는 호출할 수 없고, 관리자 권한이 있어야 합니다.

### 4. JWT 구현을 PyJWT로 교체

- `server/backend/security.py`에서 직접 구현하던 JWT 인코딩/디코딩 로직을 제거하고 `PyJWT`를 사용하도록 변경했습니다.
- 만료 토큰과 잘못된 토큰을 `PyJWT` 예외 기준으로 처리하도록 정리했습니다.
- `server/requirements.txt`에 `PyJWT` 의존성을 추가했습니다.

### 5. 최근 경기 적재 중복 요청 최적화

- Riot 적재 시 Riot에서 받은 최근 `match_id` 목록을 기준으로 DB에 이미 저장된 경기인지 먼저 확인하도록 변경했습니다.
- 기존처럼 최근 경기 목록을 간접 비교하지 않고, `match_summary.match_id` 존재 여부를 직접 조회합니다.
- DB에 이미 있는 경기는 Riot 상세 조회를 건너뛰고, 신규 경기만 Riot 매치 상세 API를 호출합니다.
- 적재 응답에 `existing_match_ids`, `pending_match_ids`, `skipped_existing_match_ids`, `riot_detail_request_count`를 포함해 운영 중 확인할 수 있도록 했습니다.

## 영향

- Riot 로더가 운영 도구로서 더 안전해졌고, 관리자만 적재 작업을 수행할 수 있게 되었습니다.
- JWT 처리가 검증된 라이브러리 기반으로 바뀌어 유지보수성과 안정성이 좋아졌습니다.
- 이미 저장된 경기에 대해 불필요한 Riot 상세 조회를 줄여 API 호출량과 적재 시간을 함께 줄일 수 있게 되었습니다.
